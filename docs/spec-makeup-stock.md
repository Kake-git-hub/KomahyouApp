# 確定仕様：振替ストック（To-Be 正本）

> 全体見直しセッション（2026-06-07）でオーナー確認済みの「目標仕様」。
> ③（月契約回数の撤廃）を受けて、振替ストックをゼロベースで再定義したもの。
> 現行実装（autoShortage 自動算出方式）からの差分が大きいため、実装はこれに合わせて作り直す。

## 1. 発生（積まれる）★再定義の核心

- 盤面の通常コマの生徒を **「欠席」登録すると、その生徒×科目の振替ストックに自動で1コマ積まれる。**
- 欠席登録した**コマは盤面から消さず「欠席」表示で残す**（出欠記録として視認できる）。同時にストック＋1。
- 欠席表示になった席は**ロックしない**。現状どおり**クリックで別生徒・体験・メモを追加できる**（席は引き続き操作可能）。
- 振替元として **日付・時限を記録**する。**理由ラベルは持たない。**
- 運用開始時の繰越は、初期設定の「未消化振替」で登録した分が最初から積まれる（`docs/spec-save-restore.md` §6）。
- **旧「自動不足分（autoShortage）」の自動計算は廃止。** 月次想定回数を前提にした残数算出はしない。

### 欠席登録UIについて
- この欠席登録UIは**未実装（README「次の実装候補1」）**。本仕様に沿って新規実装する。

## 2. 消化／戻し

- ストックを選んで**空きコマへ置くと、残が1減る。**
- 置いた振替を**盤面から外す（ストックへ戻す）と、残が1増える。**

## 3. 表示・ルール（現行から維持）

- 集計単位は **生徒 × 科目。**
- 振替ストックはコマ表操作後も**開いたまま維持**する。
- **手動追加した生徒は振替ストックにカウントしない。**
- 振替中／移動中に**同じコマへ同じ生徒が既にいると配置を拒否**し、選択状態は維持する。

## 4. 廃止する旧概念

- 自動不足分（`autoShortage`）の自動計算。
- **マイナス残（過剰配置）表示**（自動算出をやめるため不要）。

## ★ 移行（本番反映）時の必須チェック ※オーナー強指定

**既に本番運用中。反映で既存教室の振替ストック残数がズレてはいけない。**

1. 現行のストック残数（`balance` = autoShortage + manualAdjustments + 初期繰越 − 配置）を**教室ごとに事前計測**する。
2. autoShortage を廃止すると残数が変わる教室が出るため、**autoShortage 相当分を明示的な振替エントリへ確定変換（マイグレーション）**し、移行前後で各生徒×科目の残数が一致することを確認してから反映する。
3. 検証は **読み取り（GET）専用**で行う（本番教室への書き込み禁止／書き込み検証は開発用教室 `v8OZ7zH8vONNHjjYVcR1` のみ）。
4. ズレが無いことを確認できるまで本番へは反映しない。

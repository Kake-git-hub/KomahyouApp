# 確定仕様：振替ストック（To-Be 正本）

> 全体見直しセッション（2026-06-07）でオーナー確認。
> **2026-06-08 Phase 4 精査で重要訂正**：現行の `autoShortage` は「月回数−実配置」ではなく
> **「休日（休校曜日/個別休日）で潰れた通常授業を自動で振替（未消化）に計上する仕組み」**だった。
> これは有用なので**残す**（当初の「autoShortage廃止」は撤回）。

## 1. 発生（積まれる）

### A. 休日振替（自動・★残す）
- **日付をクリックして休日（holidayDates）にすると、その日に通常授業があった生徒は全員「休日振替」として未消化に自動計上**される（理由ラベル `休日振替`/`定休日振替`）。
- ★**計上タイミングは過去・未来を問わず、休日設定した時点で即時**（旧実装は「その日が過ぎてから」だったが、即時計上に変更）。
- 休校曜日（closedWeekdays）由来の潰れた通常授業も同様に振替計上。`forceOpenDates` は除外。
- 集計単位は **生徒 × 科目**。

### B. 欠席登録（明示・★新規追加）
- 盤面の通常コマの生徒を **「欠席」登録すると、その生徒×科目の振替ストックに1コマ積まれる**（休日以外の個別欠席用）。
- 欠席登録した**コマは盤面から消さず「欠席」表示で残す**。席は**ロックせず、クリックで別生徒・体験・メモを追加できる**。
- 振替元として日付・時限を記録。理由ラベルは持たない。
- ※この欠席登録UIは**新規実装**（README「次の実装候補1」）。

### C. 繰越
- 運用開始時の繰越は、初期設定の「未消化振替」で登録した分が最初から積まれる（`docs/spec-save-restore.md` §6）。

## 2. 消化／戻し

- ストックを選んで**空きコマへ置くと、残が1減る。**
- 置いた振替を**盤面から外す（ストックへ戻す）と、残が1増える。**

## 3. 表示・ルール（現行から維持）

- 集計単位は **生徒 × 科目。**
- 振替ストックはコマ表操作後も**開いたまま維持**する。
- **手動追加した生徒は振替ストックにカウントしない。**
- 振替中／移動中に**同じコマへ同じ生徒が既にいると配置を拒否**し、選択状態は維持する。

## 4. 廃止する概念

- ★**マイナス残（希望回数を超えて配置したときの過剰配置表示）は廃止**。残数は 0 を下限とし、マイナスにしない（`overAssignedRegularLessons` による減算・`negativeReason` を撤去）。
- ~~autoShortage の自動計算~~ → **残す**（§1-A）。

## ★ 実装メモ（Phase 4）

現行は仕様に近い。必要な差分は3点：
1. **A 即時計上**：`computeAutomaticShortageOrigins` の「今日まで」制限（todayKey キャップ／pastScheduledDates）を外し、休日設定した未来日も即時に振替計上する。
2. **B 欠席登録**：盤面コマの生徒を欠席登録→振替origin生成＋「欠席」表示（コマ残す・席操作可）。新規UI。
3. **C マイナス残廃止**：`balance` の `overAssignedRegularLessons` 減算と `negativeReason` を撤去し、残数を 0 下限に。

## ★ 移行（本番反映）時の必須チェック ※オーナー強指定

**既に本番運用中。本番の振替残数に直結する。**
- A（即時計上）と C（マイナス残廃止）は**意図的に残数を変える**変更。特に A は未来休日を持つ教室で残数が増える。
- 反映前に**教室ごとに変更前後の残数を読み取り専用で計測・差分確認**し、想定どおりのズレであることを確認してから本番反映する。

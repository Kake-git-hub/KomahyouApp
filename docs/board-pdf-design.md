# コマ表 PDF 設計メモ

## 目的

コマ表画面の見た目をそのまま画像化するのではなく、印刷時に読みやすい密度へ寄せた A3 縦 1 ページ PDF を安定出力する。

## 出力方針

- 出力対象はコマ表グリッド本体のみとし、振替ストック / 講習ストックの操作パネルは除外する
- 画面側の sticky 表現やスクロール状態に依存せず、全幅を展開した静的レイアウトを別 DOM に複製して描画する
- `html2canvas` で白背景 PNG を作り、`jsPDF` で A3 縦 1 ページへ配置する
- 縮尺は横幅と高さの両方から決め、1 週間の表全体を必ず 1 ページへ収める
- 生徒セルの 2 行目だけは PDF 側で文字間を詰め、必要時のみ段階的に縮小して全文表示を優先する

## レイアウト補正ルール

- グリッド幅は表示中テーブルの `scrollWidth` を基準に固定する
- ヘッダー、曜日、時間列は sticky を解除して PDF 用の静的セルに置き換える
- 時間列は 78px 幅を確保し、時限と時間帯を 1 つの縦回転ラベルへまとめる
- 時間列のラベルは太字 20px を基準にし、セル中央へ水平・垂直ともに揃える
- 教師セル、生徒セルは最小高さ 62px を維持し、一覧性を優先する
- 枠線や背景色は画面テーマではなく印刷向けの高コントラスト配色へ寄せる
- 余白は最小限に抑えつつ、縦 1 ページに収まる範囲で中央配置する
- 生徒セルの 2 行目は `通)小5算国60` のように空白を作らず連続表示し、2 行目専用の縮小で欠けを防ぐ

## メモ表示ルール

- 空席メモ文字サイズは `src/components/schedule-board/memoText.ts` の共通ロジックを使う
- PDF 側ではまず共通ロジックの初期文字サイズを適用し、2 行に収まらない場合だけ段階的に縮小する
- 追加の縮小は 4.5px を下限にし、それ未満へは落とさない
- 画面と PDF で判断基準を共有しつつ、PDF だけが必要最小限の追加縮小を許可する

## 変更時の確認観点

- 時間列がセル中央に寄り、文字が十分に大きく読めること
- スクロール位置に関係なく全列が PDF に出ること
- 振替ストック / 講習ストックの UI が PDF に混入しないこと
- メモが 2 行以内に収まり、極端なつぶれがないこと
- A3 縦 1 ページで 1 週間分が収まること
- 生徒セルの 2 行目が途中で欠けず、不要な空白が入らないこと

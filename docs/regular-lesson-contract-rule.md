# 通常授業の契約回数ルール

## 基本方針

- `月4回` は `どの月でも必ず4回` という意味ではない。
- `月4回` は `その月に有効な通常授業日が5回以上ある場合でも、生徒を配置するのは4回まで` という上限ルールを意味する。
- 月の途中から開始する場合、月の途中で終了する場合は、その月に実際に残っている通常授業日数だけを契約回数として扱う。

## 回数の数え方

- まず、その曜日・そのコマに対して、その月に存在する通常授業候補日を並べる。
- そのうち、開始日より前、終了日より後の日は数えない。
- 祝日、休校日、講師退職日などで通常授業として扱わない日は数えない。
- 残った有効日が1回から4回なら、その回数がその月の通常授業回数になる。
- 残った有効日が5回以上なら、生徒を配置するのは4回までに制限する。

## 具体例

- 月曜授業で、その月の有効な月曜が `3回` しか残っていない状態で開始する場合: `3回`
- 月曜授業で、その月の有効な月曜が `2回` しか残っていない状態で終了する場合: `2回`
- 月曜授業で、その月の有効な月曜が `4回` の月: `4回`
- 月曜授業で、その月の有効な月曜が `5回` の月: `生徒は4回まで`、`5回目は生徒を置かない`

## コマ表上の扱い

- 通常授業期間中は、祝日・休校日を除き、講師枠は毎週コマ表に残す。
- 5週ある月の5回目や、契約上その月の上限を超えた回は、生徒を置かず講師だけを残す。
- この講師だけ残る枠は、振替ストックや別の割当を入れるための空き先として使う。

## 実装上の基準

- 生徒配置の月上限は `src/components/basic-data/regularLessonModel.ts` の `capRegularLessonDatesPerMonth` を基準にする。
- コマ表生成時は、開始日・終了日で有効日を絞ったあとで月4回上限を適用する。
- 契約ルールを変更する場合は、この文書、README、開発ルール、関連テストを同時に更新する。

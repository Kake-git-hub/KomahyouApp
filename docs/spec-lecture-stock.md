## 1. 希望数はどこが正か

- 講習の希望数（科目ごとの回数）は **QR／日程表（特別講習データ）で生徒が提出した数が正**。
- 希望数の**増加は QR／日程表の提出だけ**で行う。
- **希望数の減少は盤面操作でも可能**：盤面で対象コマを**生徒クリックメニューの「削除」で希望数が −1**（ストック由来のみ・後述 §4）。
  - ★**「空にする」（日付一括）は希望数を減らさない**（2026-07-04 確定・C1／§4-2）。単純クリアで、希望数もストックも触らない。
- 生徒日程表で希望科目数を登録した時点では**自動配置されず、講習ストックに残る**。

## 2. 講習ストックの中身（残数の定義・データモデル）

- 講習ストック残 ＝ **「希望数 − 盤面に置かれた分」＋「盤面から戻した分」**（＝未配置の残数）。
- 集計単位は **講習セッション × 生徒 × 科目**。
- 対象は **countSubmitted（提出済み）かつ希望科目を登録した生徒のみ**。
  - 「通常のみ（regularOnly）」の生徒は講習ストックなし＝カウントしない。

### データモデル（2026-07-04 明文化・C2/B1〜B4）★消してはならないガード

★**振替との構造的非対称**：振替ストックは**盤面 `weeks` を走査して**配置数・消化を算出するが、**講習ストックは盤面を走査しない**。講習残数は次の「デルタ台帳」で管理する（`buildLectureStockEntries` は希望数だけを算出し、残数は `ScheduleBoardScreen` の残数計算で加減）:

1. **提出 `requestedCount`（希望数）を pending items に展開**（`buildLectureStockEntries`＝`countSubmitted && !regularOnly` の `subjectSlots` のみ）。
2. **`manualLectureStockCounts`（stockKey→整数デルタ）で加減**（B3・正負両用の1マップ）:
   - **負＝ session 由来の配置で消費した数**（配置で −1・自動割振も −1）。
   - **正＝盤面から戻した数 ＋ 繰越（初期設定）で足した数**（戻しで +1）。
3. **`manualLectureStockOrigins`（stockKey→origin メタデータ配列）**：戻し/繰越で足した分の sessionId・originDate・slot・displayName を保持。
4. **`fallbackLectureStockStudents`（stockKey→表示情報）**：未管理生徒の表示名・科目を退避（生徒が消えても一覧に出す）。

- **消化順（B2）**：session 由来の負デルタは、pending 展開の**先頭から個数分（`consumeCount`）を削る**（振替の「元日付突き合わせ＋古い順」とは**別方式**）。科目間の区別は科目別 stockKey で担保。
- **集計キー（B4）**：stockKey ＝ `生徒キー__科目__セッション`。管理生徒＝生徒ID、未管理/手動追加＝`name:表示名`。一覧のグルーピングは**生徒×セッション**（`buildLectureStockScopeKey`）で科目を束ねる。
- ★**符号（負＝消費／正＝戻し）・消化順（先頭削り）・overwrite 返却の +1 を「単純化」で触ると残数が静かにズレる**。振替の `consumeOriginDates` と同様、**消してはならないガード**。台帳が壊れても盤面から自己修復されない（振替は盤面走査＝常に盤面が真だが、講習にはその収束が無い＝C2 で reconcile は実装しない方針＝台帳を正として据え置く）。残数計算の非退行は残数ゴールデンテストで守る（主セッションが `buildLecturePendingItemsByEntryKey` 切り出し＋テストを実装）。

## 3. 配置（コマ表で置く）

- ストックを選んで**個別割振 → 選んだ空欄へ1コマ配置**。
- **自動割振 → 条件に合う候補へ順次配置**。
- 配置しても**希望数は減らない**。減るのは「未配置残数」。
- 講師日程表で登録した講師は、**対象講習期間の空き机へ自動反映**。

## 4. 戻す・削除・日付メニュー（★役割分担）

### 4-1. コマ単位の戻す／削除（生徒クリックメニュー）

- **「ストックする（戻す）」**：盤面から外し、**未配置分としてストックへ戻す**。希望数は変えない（`manualLectureStockCounts` を +1）。
- **「削除」**：そのコマを盤面から消す。**由来を問わず未配置へは戻さない**。
  - ストック由来（session）の配置を削除した場合は **希望数 −1**（＝もう要らない、の意思表示。配置消費 −1 を打ち消す +1 と `subjectSlots` −1 で純減）。
  - 手動追加（manual）の削除は希望数不変・ストックへ戻さない。
- **コマ表で手動追加した講習はストックへ戻さない**（不要なら削除）。手動追加分は希望数に含まれないため希望数も変わらない。

### 4-2. 日付クリックメニュー（★2026-07-04 確定・C1・To-Be 再定義）

日付ヘッダのメニューを**3択**に再編する:

1. **休みにする（未消化へ）** … 既存。その日の対象生徒を欠席化し、session 由来講習は未消化へ +1 戻す（§4-3・欠席と同じ）。
2. **空にする（未消化に入れず削除）** … **単純クリアで確定**（現行 `handleClearStudentsOnDate` を正とする）。**希望数 −1 もストック返却もしない**。確認文言「ストックへの移行は行いません」どおり、盤面からその日のコマを消すだけ。
3. **振替える（全員を指定日へ振替）** … **新設**（**Issue #40** で定義確定後に実装）。その日の全員を指定日へまとめて振替える操作。定義（対象範囲・元日付の扱い・講習/振替の別）は Issue #40 で確定予定。

- ★**振替側の暗黙再出現の非対称（重要）**：**振替残数は盤面走査で算出する**ため、配置済みの**振替**コマを「空にする」で単純クリアすると、その振替が消化していた origin の消化が外れ、**振替が暗黙に未消化在庫へ再出現する**（＝「空にした」のに振替残が増える）。一方**講習はデルタ台帳が残るため再出現しない**（配置消費の −1 デルタが残る）。この振替側の扱い（クリア時に origin を抑制するか＝暗黙再出現を止めるか）は **Issue #40 の「振替える」定義とあわせて確定**する。

### 4-3. `specialStockSource`（由来）× 各操作の対応表

講習コマは `specialStockSource`（`session`＝提出ストック由来／`manual`＝コマ表で手動追加）を持ち、操作ごとに挙動が分岐する（希望数・ストックに影響するのは session のみ）:

| 操作 | session（提出由来） | manual（手動追加） |
|---|---|---|
| ストックする（戻す） | `manualLectureStockCounts` +1 で未配置へ戻す | 戻さない（盤面から外すのみ） |
| 削除 | 希望数 −1（純減・未配置へ戻さない） | 盤面から消すのみ（希望数不変） |
| 欠席（休み） | 未消化講習へ +1 戻す（§4-1 の戻しと同じ） | 戻さない |
| 欠席解除 | 未消化講習を −1（戻した分を打ち消す） | 変更なし |
| テンプレ overwrite（反映日以降） | 配置済みは未消化へ +1 返却／欠席由来は相殺（§ 参照） | 返却しない |

- ★内部ヘルパの source 別分岐（`removeLecturePendingItemFromStockState`：session=−1デルタ／manual=+1側キー削除）も本表と一致させる。source の取り違えで残数が反転するため、この分岐を触るときは本表を根拠にする。

## 5. 編集・登録解除・再登録

- **講習コマのコマ表上での直接編集機能は設けない**（2026-06-09 オーナー決定）。
  - 科目・授業時間を変えたい場合は、対象コマを**「削除」し、生徒を新規追加して講習を手動追加**する（手動追加時に科目・特別講習・授業時間を選択）。
  - ※削除はストック由来なら希望数 −1（§4）。手動追加分は希望数に含まれない（が実配置＝講習回数(actual)にはカウントされる、§7）。
- 生徒日程表で登録解除すると、**その講習期間の該当生徒の講習コマを盤面から外す**。解除中は希望数が無効でストック基準にも入らない。
- 再登録すると、**解除前に配置で消費していた分も戻したうえで、希望数と一致するストックを再計算**。
- 講師日程表で登録解除すると、**その講師だけ盤面から外す**（生徒の講習予定は残る）。

## 6. 授業時間（★追加）

- 講習の**希望数提出時にも、通常授業と同じく授業時間 90／60／45分を選択できる**ようにする（提出経路＝⑦）。
- **コマ表での手動追加時にも授業時間 90／60／45 を選択できる**（2026-06-09 決定。コマ表編集の代替＝削除+手動追加の入口）。
- 未選択＝90分扱い。
- データ保持：提出由来は `SpecialSessionStudentInput.subjectDurations`（科目→分、未設定=90）。手動追加分は盤面コマの `noteSuffix`（60/45、空=90）に保持。

## 7. 表示・警告

- 盤面では**「通)」「講)」等の区分ラベル**で通常／講習を見分ける（色分けは撤廃済み・③）。
- 講師なし・制約違反・手動追加は**赤表示**、制約違反の理由はツールチップ。

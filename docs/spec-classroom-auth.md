# 確定仕様：教室権限・ログイン・開発者画面（To-Be 正本）

> 全体見直しセッション（2026-06-08）でオーナー確認済みの「目標仕様」。
> 本番は Blaze（`komahyouapp-prod`）一択。Spark 手動運用分岐は廃止しアプリ内完結へ簡略化。

## A. ログイン方式

- ログインは **Firebase のメール＋パスワード認証**。
- パスワードリセットメール送信機能あり。
  - ★表示文言は「**パスワードリセットまたはパスワード変更**」とする（動作は同じ）。
- 本番は **Blaze プラン（Functions有効）** で運用。
- `'local'` 仮ログイン（Firebase 未設定時の `authMode='local'` 分岐）は**開発時専用**として存置する
  （本番ビルドは常に firebase。撤去はしない。2026-07-04 確定・仕様監査C4）。

## B. 権限（2種類）

- **開発者（developer）**：開発者画面を持ち、**全教室を開ける／管理できる**（代理入室）。
- **室長（manager）**：**割り当てられた1教室だけ**を開ける。他教室は見えない。
- 教室が**一時停止中**だと、室長には**停止画面のみ**表示。

## C. 教室の状態

- 教室は**契約状態（契約中／解約済）**と**利用開始日／終了日**を持つ。
- 開発者は教室を**一時停止／解除**できる（停止理由付き）。
  - ★一時停止／解除は**契約中教室の全件一括のみ**。教室個別の一時停止は提供しない
    （2026-07-04 確定・仕様監査C2。将来必要になったら要望として起票）。
- **データ辞書**（語の衝突に注意・2026-07-04 仕様監査C5）:
  - `contractStatus: 'active'` ＝ 契約中 ／ `contractStatus: 'suspended'` ＝ **解約済**。
  - `isTemporarilySuspended` ＝ **一時停止**（契約状態とは別フィールド。suspended の語が重なるが別概念）。
  - 値名の整理（例 `'cancelled'` への改名）は保存データ移行を伴うため行わない。

## D. 開発者画面：教室管理（★Blase一本でアプリ内完結）

- **全教室一覧**で利用状況・契約状態・各教室の人数を確認できる。
- 「**この教室を開く**」で代理入室、「**削除**」はパスワード入力を求める。
- ★**「教室を追加」はアプリ内で完結**（Blaze の Functions が作成）。Spark の「作成済みUID貼付」分岐は廃止。

## E. アカウント管理（★アプリ内完結に一本化）

- 各教室の**管理者名・管理者メール**を編集できる。
- 管理者の**UID差し替え**（別Authアカウントへ付け替え）ができる。
- ★**新しい室長アカウントの発行・管理者削除・メール変更はすべてアプリ内で完結**する（Blaze の Functions が処理）。
  - Spark の「Firebase Console で手動」運用は廃止。

## F. 開発者のバックアップ（②と整合）

- 開発者画面に**ワークスペース全体のJSON書き出し／読み込み**がある。
- ★**「AI分析用データ書き出し」は削除**（不要）。
- **サーバーバックアップ（毎時／日次）からの復元は開発者画面に集約**（②で各教室画面からは削除）。
- ~~「Google Drive同期」「ブラウザ同期フォルダ」の設定UIは削除~~ → ★**削除は取りやめ（2026-06-08 決定）。現状維持**。
  - 理由：撤去対象が `developerCloudBackup*` で App.tsx だけで約95箇所＋保存データにフィールドがあり、撤去コスト・リスクが利益に見合わないため。現状のUIを残す。

## G. 役割分担（②と整合・確認済み）

- **室長**：自教室のみ。保存はクラウド一本化、復元は**JSON読込のみ**、1教室分のJSONバックアップ。
- **開発者**：ワークスペース全体のJSON＋サーバーバックアップ（毎時/日次）＋全教室管理。

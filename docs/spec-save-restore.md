# 確定仕様：保存・バックアップ・復元（To-Be 正本）

> 全体見直しセッション（2026-06-07）でオーナー確認済みの「目標仕様」。
> 現行実装と差分があるため、本書がこの領域の正本。実装はこれに合わせて更新する。

## 1. 保存

- 自動保存は**残す**。デバウンス **5秒**／連続操作中は最大 **20秒** でまとめて保存。
  - （現行 2秒／15秒 から変更）
- 保存先は **Firebase（クラウド）一本**。クラウドが唯一の正本。
  - ブラウザ内（IndexedDB/localStorage）への業務データ二段階保存は**廃止**。
  - 表示範囲などの UI 設定 localStorage は対象外（残す）。
- 手動「保存」ボタンは**常時表示・常に緑**。
  - 未保存あり → 押すと保存。保存済み → 「最新＝緑」のまま（押下は no-op）。グレーアウトは廃止。
- **閉じる／タブ切替（日程表タブ以外）で未保存があれば、離脱前にダイアログを出して手動保存させる。**
- 自動保存（5秒）と離脱時ダイアログは**両方残す**（二重の安全網）。

### 補足
- 自動保存の「重さ」は既に対策済み（盤面グリッド memo 化／週数の上限トリム／clone・stringify をデバウンス後へ移動）。
  自動保存自体は現状のパフォーマンス主因ではないため残す。

## 2. 保存失敗時

- **自動リトライしない。**
- JSON バックアップを**自動でダウンロードフォルダへ書き出す**。
- ダイアログで「再ログインして、ダウンロードフォルダのバックアップを読み込んでください」と作業指示を出す。

## 3. オフライン時

- **盤面を表示せず、アプリを使わせない。**
- 「オフラインのため使用できません」等の警告を出して操作をブロック。
- 想定運用：常に 1 人の室長アカウントが 1 端末でオンライン作業する。

## 4. 復元（4種類 → 1種類へ統合）

| 機能 | 扱い |
|------|------|
| 直前の Firebase 保存前へ戻す | **削除**（Undo で代替） |
| サーバーバックアップから復元 | 各教室画面から**削除**、**開発者画面のみ**へ移動 |
| ローカル自動バックアップから復元 | **削除** |
| JSON バックアップを読み込む | **残す（唯一の復元手段）** |

- 復元は「この教室だけ」に効き、他教室に影響しない。
- 復元時は確認モーダルで「元に戻せません」と警告してから実行。

## 5. JSON バックアップ（書き出し／読み込み）

- 書き出しは**開いている 1 教室分の“すべての状態・データ”**（テンプレート・設定・盤面・ストック等を含む）を 1 ファイルに。
- 読み込めば**完全復元**できること。
- ファイル名は日本語＋日時入り。

## 6. 初期設定フロー（現状維持）

- ①管理データ Excel 取込 ②机数・休校曜日 ③開始時点の未消化ストック（振替・講習）登録。
- 「初期設定を完了してコマ表をリセットする」で反映＋盤面再作成。

## 7. その他

- 危険操作後の「直前の状態に戻す」バナー（Undo）… 維持。
- 「他教室の現状データを再現（開発用教室へコピー）」… **開発者アカウント＋開発用教室の室長アカウント**で実行可（室長へ拡張）。

## 現行実装からの主な差分（実装時のTODO覚書）

1. デバウンス 2s/15s → **5s/20s**。
2. ブラウザ内二段階保存の**廃止**（クラウド一本化）。
3. 保存ボタン：disabled/グレーアウト廃止 → **常時緑**。
4. 離脱時（未保存）**ダイアログで手動保存強制**（日程表タブ除く）。
5. 保存失敗：リトライ廃止 → **JSON自動DL＋再ログイン指示ダイアログ**。
6. **オフライン時は全面ブロック**＋警告。
7. 復元 3 種（rollback／サーバー／ローカル自動）を教室画面から**削除**。サーバー復元は開発者画面へ。
8. JSON 書き出しを**教室フルスナップショット**化（テンプレ含む完全復元）。
9. 開発用教室コピーを**室長アカウントにも開放**。

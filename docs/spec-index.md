## 今回の見直しの「大方針」（横断テーマ）

1. **通常授業はテンプレート一本化**：月契約回数ルール・5週月の講師残し・年度引き継ぎ(nextStudent)を撤廃。テンプレで在籍期間中だけ毎週配置し、室長が手動調整（③）。
2. **色分け撤廃**：コマ表も日程表も種類別の色をやめ、`通)/振)/講)` のラベルで区別。注意の赤テキストのみ残す（③⑨）。
3. **保存はクラウド一本化**：ブラウザ内二段階保存を廃止。復元はJSON読込に統一（室長）。サーバーバックアップ復元は開発者画面に集約（②）。
4. **振替は欠席登録で明示発生**：autoShortage 自動算出を廃止。欠席表示でコマは残し席は操作可（④）。
5. **講習は削除＝希望−1／ストックする＝戻す**の2操作に役割分担。授業時間90/60/45を提出に追加（⑤⑦）。
6. **登録は日程表＋QRに一本化**（⑦⑨）。日程表は追従同期をやめ「最新表示」ボタンに（メモリ軽量化、⑨）。
7. **画面の削ぎ落とし**：基本データのマネージャータブ／生徒isHidden／講師の出勤可能コマ・メモ・非表示を廃止。在籍判定は日付＋高3卒業（⑥）。グループ授業機能を削除（⑧）。
8. **Blaze一本でアプリ内完結**：Spark手動運用を廃止。教室追加・管理者発行/削除/メール変更をアプリ内に。AI分析書き出し・同期フォルダ設定UIを削除（①）。

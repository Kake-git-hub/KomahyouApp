# 確定仕様：コマ表の基本配置（通常授業＝テンプレート方式）（To-Be 正本）

> 全体見直しセッション（2026-06-07）でオーナー確認済みの「目標仕様」。
> 旧 `docs/regular-lesson-contract-rule.md`（月4回ルール）は**廃止**。本書がこの領域の正本。
>
> ## ★ 実装状況（2026-06-08 Phase 3 検証）
> 精査の結果、**本仕様の核心挙動は既に実装済み**だった：
> - 通常授業の生成元は `buildRegularLessonsFromTemplate`（テンプレ）に一本化済み。盤面・履歴・日程表とも。
> - 月契約回数（`capRegularLessonDatesPerMonth`）は呼び出し0件＝未使用。配置・想定回数(`buildExpectedRegularOccurrences`)とも月上限slice無し。→ **本Phaseで死蔵関数 `capRegularLessonDatesPerMonth` を削除**。
> - 在籍期間中だけ毎週配置（配置・想定回数とも `isActiveOnDate`＝入塾/退塾/高3卒業 でフィルタ）。
> - 5週月の講師残しロジック無し（月上限が無いため毎週フル配置）。
> - 基本データの通常授業タブは既に無い。
> - `nextStudent*` はテンプレ生成で常に空。型フィールドのみ死蔵（今回は据え置き）。

## 1. 通常授業は「通常授業テンプレート」だけで管理する ★方針転換

- 通常授業の唯一の正＝ **通常授業テンプレート**（`RegularLessonTemplate`）。
  - 構造：`cells[]` = 曜日 × 時限 × 机（`desks[]`）。各机に講師1人＋ペア生徒最大2人＋科目。
- 基本データ画面の旧「通常授業（`RegularLessonRow`）」セクションは**撤去候補**。
  - どこかで必要なら参照用バックデータとして残すかは実装時判断。コマ表配置の根拠にはしない。

## 2. 配置ルール（毎週・在籍期間のみ） ★核心

- テンプレートに登録された机・生徒を、**毎週、該当する曜日・時限へそのまま配置**する。
- 各生徒は**在籍期間中の週だけ**表示する（入塾日〜退塾日の範囲内のみ）。
- **月◯回といった契約回数の上限・カウントは持たない**（＝旧「月4回ルール」を完全撤廃）。
  - 回数調整が要るときは**室長がコマ表上で手動調整**する。
- **年度替わりの引き継ぎ（旧 nextStudent 欄）も廃止**。年度替わりもテンプレートを編集して管理する。

### 撤廃する旧概念（まとめ）
- 月4回など月次上限（`capRegularLessonDatesPerMonth`）
- 有効日カウント／想定回数ベースの配置制限
- 5週月の「5回目は生徒を置かず講師だけ残す」枠
- nextStudent（次の生徒引き継ぎ）

## 3. 休み（休校・休日・強制開校）

- **休校曜日**（既定：日曜）はコマを作らない。
- **個別の休日（holidayDates）** もコマを作らない。
- **強制開校日（forceOpenDates）** は休校曜日でもその日はコマを作る。
- **祝日は特別扱いしない＝平日と同じく毎週配置する**（祝日データは持たない）。

## 4. 1コマの構成

- 1コマ＝「曜日 × 時限」。**講師1人＋生徒最大2人（ペア着席）**。
- 時限は 1〜5限。机数は教室設定（既定14）。
- 学年度は4月始まり（4/1〜翌3/31）。
- ペア生徒は**それぞれ別科目・別在籍期間**を持てる。

## 5. 表示・色分け・警告

- **コマの種類別の色分けは撤廃**（通常/振替/代行/講習の色分けはしない）。
  - ※ 振替・代行・講習という概念・操作自体は残る。色で区別しないだけ。
- **「注意付き」テキストの赤表示は残す。**
- **手動追加した生徒は赤表示＋ツールチップ「手動追加」。**
- 講師なし・制約違反・手動追加は赤表示の対象、詳細理由はツールチップ。

## 6. 授業時間

- 各授業に**授業時間 90／60／45分**を持てる（未選択＝90分扱い）。
- **【2026-07-04 確定・オーナー確認済み／C1：データ辞書明記（専用フィールド化しない）】**
  - 通常授業の授業時間は**専用フィールドを持たず**、`RegularLessonTemplateStudent.note`
    （盤面では `noteSuffix`）を**転用**して保持する。値は `normalizeRegularLessonNote`
    （`src/components/basic-data/regularLessonModel.ts`）で `'' | '60' | '45'` に正規化し、**`''` ＝ 90分扱い**。
  - **この `note` を「授業時間」以外の用途（純粋なメモ等）に流用してはならない**。流用すると保存済みの授業時間が
    失われて回帰する（転用の意図を変更しない・消してはならない）。
  - 講習（special）側は**専用フィールド** `lectureDurationOptions=[90,60,45]`
    （`src/components/special-data/specialSessionModel.ts`）を持つ。通常授業（note 転用）と講習（専用フィールド）で
    保持方法が**非対称**である点に注意（機能としてはどちらも 90/60/45 を保持・表示できる）。

## 7. テンプレート挙動の細部（凍結・保存・生成範囲・編集）— 2026-07-04 確定・オーナー確認済み（C2）

> **テンプレ挙動の細部の正本は `docs/spec-template-behavior.md`（Q1-Q20）**。本書は上位方針を、
> 同書は反映日境界・凍結・休日との力関係・overwrite 時のストック返却/相殺・生成範囲・履歴などの
> Q&A 詳細を定義する。**両書は補完関係**であり、片方だけを編集して齟齬を出さないこと。以下は同書の要点を
> 上位正本にも取り込んだ要約（詳細と根拠 file:line は同書を参照）。

### 7-1. テンプレ編集の本体経路（C3）
- テンプレ編集の本体は**オンボード経路**（`ScheduleBoardScreen` の `isTemplateMode`/`templateCells`。盤面を週グリッドで
  編集し、保存は `handleSaveRegularLessonTemplate`）。モーダル版 `RegularLessonTemplateEditor.tsx` は**どこからも import されて
  いない死蔵コンポーネント**で、**削除する**（削除実装は別途）。テンプレ挙動の変更は必ずオンボード経路に入れる。

### 7-2. 反映日と凍結（`templateFreezeBeforeDate`）— B2・消してはならないガード
- テンプレ反映日（`effectiveStartDate` ＝保存時に `templateFreezeBeforeDate` へ設定）**より前**のコマ表は、テンプレ再生成で
  **一切変更しない**（講師・生徒・出欠・メモすべて不変）。再構築は反映日**以降**のみ。
- 盤面再生成（`createBoardWeek`＋overlay）は `templateFreezeBeforeDate` 未満の週/セルの managed overlay を完全スキップする。
- **これは週トリミング（`boardWeekTrim`：範囲外の未編集週＝テンプレ再生成で同一だから破棄）の安全前提でもある。**
  凍結ガードが外れると反映日以前の確定済みコマ（過去の実績）が再生成で書き換わる/消える危険があるため、**消してはならない**。

### 7-3. overwrite 保存時のクリアとストック返却/相殺 — B3・破壊的挙動
- テンプレ保存（overwrite）時は**反映日以降の全セルを一旦クリア**（講師・手動講師・自動割振由来・メモ・出欠 status・授業を消去）
  してからテンプレ通常授業を再配置する。ユーザー視点では保存モードは1つ（overwrite/定義のみ保存の区別は見せない）。
- クリア前に、反映日以降に**配置済みだった振替(makeup)/講習(lecture)は未消化ストックへ返却**（Q7）。欠席由来で生まれていた
  **未消化振替/講習は再構築で相殺**（二重計上防止・Q8）。**手動追加した振替は復元しない**（Q9）。反映日以降ぶんの
  手動振替調整・各種抑制(suppressed)・希望回数補正(scheduleCountAdjustments) は**クリア**、反映日より前ぶんは**保持**（Q10）。
- **室長の体感に直結する破壊的挙動**（再保存でコマ表の見た目が大きく変わり、振替/講習がストックへ戻る）。意図せぬ再保存に注意。

### 7-4. 生成範囲は単年度のみ — B4
- テンプレは**反映日〜その年度末（3/31）の単年度だけ**生成する（複数年一括・上限年度は撤廃・Q11）。
  年度替わりは**必ず新テンプレを作成**して管理する（履歴は最新3件まで保管・Q16）。

### 7-5. テンプレ編集時の配置対象 — B5
- **テンプレ編集モードでは「入会前」＋「在籍中」の生徒/講師を配置できる**（入会前＝未来開始も先行配置可・退塾のみ除外・Q14）。
  ただし**実際の盤面表示は在籍開始週から**（配置後の実盤面は §2 どおり在籍期間のみ表示）。矛盾ではなく段階の違い。

### 7-6. テンプレ編集モードと休日の力関係 — B6
- **テンプレ編集モードでは定休日（`closedWeekdays`）セルに講師・生徒を配置できない**（混乱防止・Q4）。
- テンプレが消すのは **`holidayDates`（手動の個別休日）だけ**。**`closedWeekdays`（定休日）と `forceOpenDates`（強制営業日）は
  テンプレでも消さない**（「テンプレ ＜ 定休日」「個別休日 ＜ テンプレ」の非対称）。反映後に付けた休日は生きる（Q5）。

## ★ 下流ブロックへの影響（要再設計・覚書）

月契約回数を撤廃したことで、回数を前提にしていた機能の定義を作り直す必要がある：

1. **振替ストック（ブロック④）**：従来は「通常授業の月次想定回数 − 実配置」で残数を出していた。
   月次回数が無くなるため、振替ストックの発生・残数の定義を作り直す（例：欠席1回＝振替1コマ、手動管理 等）。→ ブロック④で確定する。
2. **生徒日程表の通常回数（ブロック⑨）**：actual/planned カウントの基準が変わる。
   「想定回数」＝在籍期間内の週次出現数ベースへ再定義。
3. **撤去対象コード（実装時）**：`capRegularLessonDatesPerMonth` / `buildExpectedRegularOccurrences` /
   `buildScheduleCellsForRange` の月次上限分岐 / `RegularLessonRow.nextStudent*` / 色分けロジック。
4. **ゴールデンスナップショットテスト**：boardSnapshot / makeupStockSnapshot / scheduleOccurrences を作り直す。

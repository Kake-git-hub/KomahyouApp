# 確定仕様：基本データ画面（To-Be 正本）

> 全体見直しセッション（2026-06-08）でオーナー確認済みの「目標仕様」。
> ③（通常授業＝テンプレ方式）と整合。現行から項目を削ぎ落とす方向。
> **2026-07-04 監査（領域6）でオーナー確定**：C1〜C3・B2〜B6 を明文化（末尾「## 監査確定（2026-07-04）」参照）。

## A. タブ構成

- タブは **生徒／講師／教室データ** の3つ。
  - ★**マネージャータブは削除**（使用していない）。
  - ★**managers はデータ配管ごと完全撤去する**（2026-07-04 確定・**Issue #41**）。撤去方式は「自然消滅」＝ロード時は tolerant に無視、保存時に書き出さない（本番一括書込なし）、旧「マネージャー」Excel シートを再取込しても無視。**撤去完了までは型・保存・Excel シートが残存する（現状）**。
- **並び替え・フィルタは生徒・講師タブの表に限る**（教室データタブは定休日チップ＋机数の設定フォームで表ではない）。
- ペア制約は自動割振ルール画面で管理（⑧）。

## B. 生徒タブ

- 入力項目：**名前・表示名・メール・入塾日・退塾日・生年月日**。
  - ★**「非表示（isHidden）」項目は廃止**（手動の非表示フラグは持たない）。
- 学年は**生年月日から自動判定**（学年欄は持たない）。
  - ★**学年ラベルの定義（2026-07-04 明文化・B2）**：生年月日×基準日で **未就学／小1〜6／中1〜3／高1〜3** を算出（`resolveGradeLabelFromBirthDate`）。
    - **学年度境界は 4/1**（基準日が 4/1 以降ならその年、未満なら前年を学年度とする）。
    - **早生まれ（1〜3 月生）は1学年上**に数える（`resolveEnrollmentYearFromBirthDateParts`：4月未満は誕生年+6・4月以降は+7 で入学年度を出す）。
    - 入学年度前は「**未就学**」。**高3（学年番号12）の学年度末（3/31）を過ぎると卒業＝非在籍**（`hasGraduatedHighSchool`＝学年番号≥13）。
- 在籍／非在籍は**自動判定**：
  - **入塾日より前・退塾日より後・高3卒業後**は「**非在籍**」として扱う。
  - ★**高3卒業後は自動的に非在籍**（卒業判定を生年月日から行う）。
- 非在籍の生徒は盤面・各種集計から外れる。

## C. 講師タブ

- 入力項目：**氏名・表示名・メール・入塾日・退塾日・担当科目（科目＋担当上限学年）**。
  - ★**「出勤可能コマ」「メモ」「非表示(isHidden)」は廃止**（出勤枠はテンプレで管理するため不要）。
- ★**講師の休職などで一時的に隠したい場合は「退塾日」で代用**する（2026-06-08 決定C）。復帰時は退塾日を戻す。
  - 在籍判定の共有関数(isActiveOnDate/resolveScheduledStatus 等)から isHidden を外すため、講師も isHidden を持たない。
- 担当科目は「**選んだ学年以下を担当可能**」（上限学年方式）。
- **在籍講師／非在籍講師の表示切替**がある（非在籍は入塾日・退塾日から自動判定）。

## D. 教室データタブ

- 設定するのは **定休日（休校曜日）と机数**。
- 役割分担：
  - **初回は「初期設定フロー」（②）で設定**。
  - **途中変更はこの「教室データタブ」で行う**（両方残す）。

## E. 通常授業（③の整合）

- 基本データに**通常授業の編集タブは無い**。通常授業は**コマ表のテンプレモード**で編集。
- Excelには「**通常授業テンプレ**」シートがあり、テンプレを書き出し・取り込みできる。
- ★**通常授業モデル `RegularLessonRow` の注記事項（2026-07-04 明文化・B5/B6）**：
  - **年度引き継ぎフィールド `nextStudent1Id/nextSubject1/nextStudent2Id/nextSubject2` は死蔵**（常に空・領域3 §2 で引き継ぎ廃止確定）。将来除去候補（除去時は保存データ互換に注意）。
  - **`student1Note/student2Note` は授業時間（''=90／'60'／'45'・`normalizeRegularLessonNote`）の転用**であり、**メモに流用しない**（領域3 §6 と同根＝流用すると授業時間が失われ回帰）。

## F. Excel取込・出力（実体は「バックアップ/復元/初期設定」画面）

- 管理データ（生徒・講師・教室データ・通常授業テンプレ）を**テンプレ出力／現データ出力／初期取込／差分取込**できる。
- 差分取込は**ID列を残して再取込すると、講習・ルール・盤面・ストックを保持したまま更新**。
- ★**同一性照合の順序（2026-07-04 明文化・B3）★消してはならない設計**：再取込は **ID → メール → 表示名 → 名前** の順で既存行に照合し、一致すれば**既存 ID を維持**する（`findStudentMatch`／`findTeacherMatch`）。**改名しても ID/メール/表示名で追随**できる。
  - ID を保持するのは、盤面・振替/講習ストック・生徒日程表が生徒を **`managedStudentId` で参照**するため。照合順を「ID のみ」等に単純化すると、ID 空欄 Excel の再取込で全生徒が新規扱いになり盤面から生徒が消える／二重化する。memory `komahyou-no-regression-rule` の `mergeManagedDeskLesson` 同一性補完と**同根の消してはならない設計**。
- ★**差分取込は更新・追加のみ（2026-07-04 明文化・B4）**：取込ファイルに無い既存生徒/講師は**削除されない**（`mergeImportedBundle` は更新/追加のみ）。削除は基本データ画面から個別に行う。取り違えで大量削除が起きない安全側設計。
- ★Excel運用は**初回セットアップで必要なので残す**が、③のテンプレ方式に合わせて**シート構成を整理**する。**整理の完了条件（2026-07-04 確定・C3／差分5）**：
  - **残すシート構成＝講師／生徒／教室データ／通常授業テンプレ**（「マネージャー」シートは Issue #41 で撤去）。
  - 死蔵フィールド `nextStudent*`（`RegularLessonRow`＝§E）は将来除去候補（除去時は保存データ互換に注意）。

## G. 祝日・強制開校日

- ★**holidayDates（個別休日）／forceOpenDates（強制開校日）は「盤面」で設定する**方針。
- **基本データ画面には祝日・強制開校日の編集UIは一切置かない。**

## H. 生徒／講師の削除（2026-07-08 オーナー確定）

- 削除は**確認モーダル**で行う（`window.confirm` は廃止）。モーダルには必ず次を表示する（`buildDeleteConfirmation`／`deleteGuard.ts`）:
  - ★**「削除したデータは元に戻せません」**の不可逆警告。
  - ★**「データを残したいだけなら、削除せず退塾日を設定して非表示にできる」**案内。手動 isHidden は §B/§C のとおり**復活させない**（隠すのは退塾日で代用が確定仕様。2026-07-08 に再確認）。
  - ★**未消化の講習／振替が残っている生徒**を削除しようとした場合は、その残数を明示して警告する。残数は盤面（`ScheduleBoardScreen`）が算出したものを親（App）へ持ち上げて突き合わせる（`onDeletionStockSummaryChange`）。
- ★**削除実行にはログインアカウントのパスワード再認証を要求**する（本番=firebase のみ。ローカル開発はログインアカウントが無いため不要）。教室削除（`deleteClassroom`）と同じ `reauthenticateFirebaseUser` を用いる。
- 背景：講習提出済みの生徒（富樫兄弟＝ID `s091`/`s129`）を名簿から削除し、提出（`studentInputs`）が孤児化して講習一覧に生ID表示された事象（2026-07-08）の再発防止。

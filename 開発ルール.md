# 開発ルール

## 目的

コマ表、振替ストック、日程表、講習期間表示の変更で回帰を残さないため、実装修正後に定期的な確認を必ず実施する。

## 実施タイミング

- コマ表の配置ロジック、振替ロジック、休日処理を触った直後
- 生徒日程表 / 講師日程表 / PDF / 講習期間帯を触った直後
- リリース相当の区切り前
- 少なくとも週 1 回の保守確認時

## 必須チェック

1. `npm run build`
2. `npx playwright test tests/schedule-board.spec.ts`

## Playwright で最低限守る観点

- 基本データの追加、編集、削除がコマ表へ反映されること
- 期間付き通常授業の契約回数は開始月 / 終了月を含む月数 × 4 として扱われること
- 振替ストックの計算、消化、再ストックが崩れないこと
- 振替中 / 移動中に同コマ重複を拒否し、選択状態を維持すること
- 日程表タブが別タブ表示、日付入力即反映、講習期間セレクター反映、ライブ追従更新を維持すること
- 通常授業を stock へ回した未割当分が日程表に残らないこと
- 通常回数 / 講習回数の警告スタンプがそれぞれの表の直下に出ること
- 生徒日程の `算` / `数` が学年に応じて片方だけ表示されること
- 講習期間帯と休日グレーが表示されること

## ドキュメント更新ルール

- 仕様や運用ルールを変えたら `README.md` と `.github/copilot-instructions.md` を同じ変更内で更新する
- 日程表の反映ロジックや stock 表示ルールを変えたら `docs/schedule-popup-design.md` も更新する
- テスト運用や確認手順を変えたらこのファイルも更新する

## 失敗時の扱い

- build または Playwright が落ちた状態で作業完了扱いにしない
- 既知の失敗として残す場合は、失敗テスト名、原因、暫定回避、次の対応を記録する
